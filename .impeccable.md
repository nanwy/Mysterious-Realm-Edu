## Design Context

### Users

云学考系统当前首期重点面向学员端用户。用户使用产品时，不只是为了完成单一任务，也会在学习、考试、平台动态、推荐内容和个人事务之间来回切换。界面需要同时承担四类职责：

- 内容分发
- 任务推进
- 状态感知
- 平台节奏表达

这不是一个只服务“继续学习”的单点工具，也不是一个纯内容展示站。学习、考试、资讯、通知和个人事务必须被组织成同一产品语言，而不是被做成多个割裂的子系统。

### Brand Personality

品牌应长期呈现为：

- 年轻
- 锐利
- 精致

整体气质不是传统教育官网，也不是平庸的 SaaS 工具站，而是更像新一代创意数字产品与学习考试系统的结合体。界面应持续传达这些感受：

- 未来感
- 效率感
- 探索欲
- 平台势能

用户进入页面后，应该感到“这个系统很新、很强、很有内容，而且不像普通教育站点”。

### Aesthetic Direction

视觉方向采用年轻创意数字产品风格，而不是“高端科技平台”的空洞炫技路线。整体应更明亮、更数字化、更像精致的 macOS 原生工具。长期视觉语言应优先依赖：

- 高对比排版
- 考究留白
- 极细分割线
- 克制但高级的阴影
- 冷金属或切面式表面
- 强网格秩序
- 有节奏的微交互

整体建议同时设计 light mode 与 dark mode，两者都必须成立，但不能互为简单反色，而且要故意走出不同气质：

- Light mode：
  - 更接近 Next.js 官网式的黑白灰排版控制
  - 更接近冷中性、切面、拼接、金属感
  - 结构优先，材质其次
  - 不允许退回“白底 + 大圆角白卡片 + 单一主色点缀”

- Dark mode：
  - 更接近 Linear 式黑色高级工具感
  - 更冷、更硬、更克制
  - 靠边界、表面和层级成立
  - 不是普通深色后台
  - 不是蓝黑 AI 渐变
  - 不是廉价霓虹赛博风

light mode 允许混合三种材料语言：

- 冷金属面板
- 玻璃磨砂
- 纸面式排版

但要有明确主次，不能平均使用。优先级始终是：

1. 信息节奏
2. 排版控制
3. 结构关系
4. 材质特效

参考产品的视觉逻辑应优先吸收：

- Linear：专业工具感、阴影层级、收敛但锐利的交互反馈
- Reflect：磨砂玻璃、流动卡片、个人驾驶舱式的精致感
- Vercel / Next.js 官网：黑白灰与极细网格、开发者审美、技术先进性

### Anti-Direction

以下方向属于长期禁区：

- 传统政务或学校后台风
- 招生官网式大图、金刚位、运营感 banner
- 千篇一律的通用 SaaS Landing Page
- 大横幅 hero
- 左文右图模板首页
- 一排排相同卡片
- 模块标题 + 查看更多 的默认 section 语法
- 白底大圆角白卡片拼盘
- 统一阴影卡片解决一切
- 蓝紫主色点缀一切的保守工具站
- 普通深色后台主题
- 典型 AI 产品页配色和组件语言

### Design Principles

1. 产品页面首先要表达“顺序”和“重点”，而不是只罗列入口。平台应替用户做一部分优先级判断，而不是把所有内容平权摆出来。
2. 学习、考试、资讯、通知和个人事务必须属于同一产品语言，不能被做成割裂的子系统。课程、考试、动态和推荐应被视作同一信息流中的不同信号层。
3. 允许非对称版式、强主次、窄列高密度和不平权模块。不要为了整齐而牺牲势能，重要内容必须获得更大面积、更强对比和更高节奏控制权。
4. 设计应更多依赖连续版面、共享边界、分区拼接和信息密度差异，少依赖独立大容器、完整盒子和规则网格平均分块。
5. 标题不能只是“大”，而要有明确的权力结构。页面阅读顺序应尽量由排版密度、栏宽、层级和位置决定，而不是靠用户自己扫。
6. 文案应该像产品在说话，不像设计师在解释页面。少写设计意图，多写真正帮助用户判断和行动的话。
7. 色彩应服务层级、状态和节奏，而不是承担主要设计工作。颜色使用必须克制，但不能无聊。中性色应带冷色偏移，避免纯灰、纯黑、纯白的生硬感。
8. 组件应服从版面，不应反过来主导版面。去卡片化优先级很高；如果一个区域过于完整、过于像独立模块，它就会重新把页面拉回“模块组合页”。
9. 无数据、预备态、同步中状态必须像产品的一部分，不像系统调试提示。错误态不能泄露“开发中”感，占位内容也必须自然融入版面。
10. 所有新设计优先通过语义 token 实现，例如 `background`、`card`、`foreground`、`muted-foreground`、`primary`，避免依赖固定调色板类。组件尽量复用 `packages/ui` 中的 shadcn/ui。
11. 桌面端不是把小屏简单放大。高频控制区、筛选区、表单区在桌面端默认横向编排和底部对齐，小屏下再纵向化。
12. 任何新设计都应用这些问题自检：
    - 它像平台，还是像模块集合？
    - 它像工具产品，还是像模板站点？
    - 它的主次是结构决定的，还是颜色和卡片硬撑的？
    - 它是在替用户排序，还是把决策成本甩给用户？
    - 它像真实产品语言，还是像设计说明和占位文案？
