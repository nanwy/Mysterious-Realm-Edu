## Design Context

### Users
云学考系统当前首期重点面向学员端用户，但首页定位不再只是个人学习驾驶舱，而是一个更强的平台内容门户。用户进入系统时，不仅需要快速继续课程、练习或考试，也需要浏览平台重点内容、教务动态、考试安排、推荐课程与学习任务。界面需要同时承担“内容分发”和“任务导航”两种职责，让用户一眼感知平台活力、重点入口与当前节奏。

### Brand Personality
品牌应呈现为：年轻、锐利、精致。整体气质不是传统教育官网，也不是平庸的 SaaS 白卡片后台，而是更像新一代创意数字产品与学习考试系统的结合体。界面应传达的核心感受是：未来感、效率感、探索欲、平台势能。用户进入页面后应感到“这个系统很新、很强、很有内容，而且不像普通教育站点”。

### Aesthetic Direction
视觉方向采用年轻创意数字产品风格，而不是“高端科技平台”的空洞炫技路线。整体应更明亮、更数字化、更像精致的 macOS 原生工具：强调高对比排版、考究留白、极细分割线、克制但高级的阴影、半透明磨砂层、强网格秩序和有节奏的微交互。首页应像一个数字内容与学习任务的中控界面，而不是营销落地页或教务公告板。

整体建议同时设计 light mode 与 dark mode，两者都必须成立，但不是简单反色。Light mode 应偏冷光、玻璃感、数字桌面感；dark mode 应偏深空、电蓝、荧光、高对比信息面板感。视觉焦点应围绕课程入口、考试安排、内容推荐和平台动态展开，形成更具未来感的门户首页。

参考产品的视觉逻辑应优先吸收以下方向：
- Linear：专业工具感、阴影层级、收敛但锐利的交互反馈
- Reflect：磨砂玻璃、流动卡片、个人驾驶舱式的精致感
- Vercel / Next.js 官网：黑白灰与极细网格、开发者审美、技术先进性

明确的反方向包括：
- 传统政务或学校后台风
- 招生官网式大图和金刚位
- 千篇一律的通用 SaaS Landing Page

### Design Principles
1. 首页必须像“平台内容门户”，而不是样板化的营销首屏或后台仪表盘。要有内容密度，但仍保持秩序和呼吸感。
2. 学习平台与考试产品要融合成统一产品语言，像一款创意数字工具，而不是多个业务模块的拼盘。
3. 未来感来自排版、网格、层级、切面、磨砂、细线和系统性微交互，不来自廉价插画、伪科技地球、芯片或泛蓝色渐变。
4. light 与 dark 都要作为一等视觉系统来设计，且两者都应具备独立成立的美学，不做简单反色。
5. 所有新设计都优先通过语义 token 实现，例如 `background`、`card`、`foreground`、`muted-foreground`、`primary`，避免继续依赖固定调色板类。
6. 组件尽量复用packages/ui中的shadcn/ui。
